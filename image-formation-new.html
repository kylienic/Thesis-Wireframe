How a Digital Image is Captured



Light enters a camera through a glass optical lens. The lens then focuses the incoming light onto a mirror, where it is then reflected onto the picture plane.

The image is focused directly onto the camera sensor, which records the incoming light. The sensor consists of thousands, if not millions, of light-sensitive photosites which accumulate a charge from the incoming light. Each photosite accumulates a charge, which it then outputs in the form of a voltage waveform. This initial charge is a physical (or analog) measurement. Before it can be processed, it must be converted to a digital signal, which will be discussed in the next section.

Photosites are atomic picture sensing elements which capture continuous light in discrete samples. The magnitude of the charge depends on the strength and duration of light exposure. For example, brighter portions of an image result from higher-charged photosites than their dimmer companions. Photosites are arranged in rows and columns on a flat rectangular sensor- they exist in a two-dimensional array.

A typical sensor consists of 4000x4000 photosites or more. Each photosite corresponds to a pixel in the resulting image, so image size and resolution depends greatly on the size and quality of the image sensor. Image pixels are also arranged in two-dimensional arrays.

Math fact: A camera sensor produces a digital output proportional to the integral of the light received at each sensor.

Once recorded, the individual analog charge magnitudes are digitized: converted to discrete numerical quantities. The analog signal is continuous, while the digital signal (by definition) is discrete. In order to convert the analog signal to digital, the analog signal is sampled (measured at regular intervals) and then quantized (assigned to a set of integer values). This process occurs within an analog-to-digital circuit.

During quantization, each sampled value is assigned an integer representation. Pixel intensity ranges from 0 to 255, and so each value must be mapped to the integer value that most closely represents it. The result is stored in an array. A digital image is, therefore, a sampled collection of integer values that represent light recorded by the image sensor.

The image array can have any number of operations performed on it, which is known as digital image processing. A common operation performed during quantization is noise reduction, which consists of averaging image data. This is performed by an Arithmetic Logic Unit, which performs arithmetic and logical operations on entire images in parallel.

If a camera is not storing RAW (or uncompressed) image data, it must convert the data to a smaller image file format. Image file formats include JPEG, PNG, GIF, among others. These conversions require compression of image data, which can result in diminished image quality. However, compression is necessary for fast transmission of images across web-enabled devices, as well as for mass storage of images. Conversion is performed in the image processing stage, before the image is stored onto long-term media storage. Often a camera will apply a tone curve, or light contrast adjustment to an image before it is converted and saved in JPEG or PNG format.

Finally, the RAW image data can be stored onto a memory card, or the processed, curved and JPEG-converted image can be stored. This data can be accessed later by a computer for viewing, sending, and/ or editing.

A note on file size/ storage: 1024x1024 image at 8bits per pixel is 1MB storage if image is uncompressed. A typical JPEG is 3MB, while a RAW file can be upwards of 43 MB.
